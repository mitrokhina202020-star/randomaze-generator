Ты помогаешь работать с проектом **Randomaze Generator** — инструментом для генерации рандомайзера учебных проектов.

## Структура проекта

```
randomaze_generate/
├── 2026-06-15_randomaze-gen_page.html   ← единственный рабочий файл
├── README.md
└── .claude/commands/randomaze.md
```

Весь функционал — в одном HTML-файле `2026-06-15_randomaze-gen_page.html`. Никаких зависимостей, библиотек и базы данных нет.

### Что внутри файла

| Блок | Что делает |
|------|------------|
| HTML/CSS (верх) | Интерфейс: карточки проектов, поля настроек, кнопки |
| `addCard()` | Добавляет карточку проекта в редактор |
| `importFromFile()` | Парсит Notion HTML-экспорт, создаёт карточки из тогглов |
| `sanitizeHtml()` / `walkNodes()` | Очищает HTML при вставке из буфера |
| `generate()` | Собирает данные из карточек и запускает генерацию |
| `buildOutput()` | Строит полный рандомайзер (HTML + JS) с историей и кнопкой «Сохранить» |
| `buildSimpleOutput()` | Строит упрощённый JS-скрипт (только названия, реролл) |

## Как запустить локально

Файл — чистый HTML без серверной логики. Открыть напрямую в браузере:

```bash
open 2026-06-15_randomaze-gen_page.html
```

## Как вносить изменения

Все изменения — только в `2026-06-15_randomaze-gen_page.html`. После правок:

1. Проверить файл в браузере (`open 2026-06-15_randomaze-gen_page.html`)
2. Обновить README.md если изменилось поведение
3. Закоммитить и запушить:

```bash
git add 2026-06-15_randomaze-gen_page.html README.md
git commit -m "описание изменений"
git push
```

## Как обновить README

README описывает инструмент с точки зрения пользователя. При обновлении проверить:
- Раздел «Возможности» актуален?
- Схема «Как работает» соответствует коду?
- Имя файла в разделе «Использование» не изменилось?
- Ссылка на GitHub Pages рабочая?

## Проверка ошибок

Типичные проблемы:

| Симптом | Где смотреть |
|---------|--------------|
| Импорт из Notion не подхватывает проекты | `importFromFile()` — проверить селектор `details` и `.indented` |
| Генерация выдаёт пустой код | `generate()` — проверить, заполнен ли `lesson-id` и есть ли карточки |
| Кнопка «Сохранить» не срабатывает | `buildOutput()` — проверить SAVE_ENDPOINT и CORS на принимающей стороне |
| Форматирование ломается при вставке | `walkNodes()` — проверить обработку тега, который ломается |
